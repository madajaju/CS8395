= API Gateway vs. Service Mesh
:revdate: 2026-03-24
:revealjs_theme: black
:revealjs_transition: slide
:source-highlighter: highlightjs
:icons: font

== Purpose
Differentiates between "North-South" (edge) and "East-West" (internal) traffic management.

== Outcomes
- Contrast API Gateway and Service Mesh responsibilities.
- Identify common features (Rate Limiting, mTLS, Load Balancing).
- Evaluate when a service mesh adds too much complexity.

== Overview
Managing traffic is a first-class concern in distributed systems.
- **North-South Traffic:** External traffic from users to your system (Edge).
- **East-West Traffic:** Internal traffic between your services (Internal).

== API Gateway (The Edge)
Acts as a single entry point for all clients.
- **Primary Roles:** Auth, rate limiting, request routing, protocol translation, request aggregation.
- **Focus:** User experience and security at the perimeter.

== Service Mesh (The Mesh)
Dedicated infrastructure layer for service-to-service communication.
- **Primary Roles:** mTLS, retry logic, circuit breaking, observability.
- **Implementation:** Usually via "Sidecar Proxies" (e.g., Envoy).

== Comparison Table
| Feature | API Gateway | Service Mesh |
| :--- | :--- | :--- |
| **Location** | Perimeter (Edge) | Internal (Sidecar) |
| **Traffic Direction** | North-South | East-West |
| **User-Facing** | Yes | No |
| **Granularity** | Coarse (Global) | Fine (Instance-level) |
| **Complexity** | Medium | High |

== Code Example: Java (Spring Cloud Gateway)
[source,java]
----
// Coarse-grained routing at the edge
@Bean
public RouteLocator customRouteLocator(RouteLocatorBuilder builder) {
    return builder.routes()
        .route("path_route", r -> r.path("/get")
            .uri("http://httpbin.org"))
        .build();
}
----

== Code Example: YAML (Envoy Retry Filter)
[source,yaml]
----
# Simplified Envoy filter for internal retries
retries:
  retry_on: "connect-failure,5xx"
  num_retries: 3
  per_try_timeout: 2s
----

== Code Example: Node.js (Manual Retry)
[source,javascript]
----
// Without a mesh, service must handle its own retries
const response = await axios.get('http://inventory-service/items', {
    timeout: 5000,
    retries: 3 // Manual logic
});
----

== Risks and Tradeoffs
- **Latency:** Each proxy hop adds milliseconds.
- **Complexity:** Service meshes require complex control planes (e.g., Istio).
- **Reliability:** Gateway is a central point of failure if not HA.
